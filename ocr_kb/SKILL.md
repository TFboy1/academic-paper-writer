# 论文排版与整理完全工作流 (Iterative OCR & Typesetting Pipeline)

## 0. 核心架构理念与约束 (Core Workflows)
当用户要求进行长论文解析和排版时，你必须**严格按顺序**执行以下三大核心步骤。禁止擅自跳跃或合批。

### 第一步：逐页精读与追加写入 (Page-by-Page Extraction)
对于切分好的每一张长文档单页图片，你需要：
1. **纯文本读取**：一页一页读取，将文字原汁原味地提取出来。
2. **公式转译**：如果页面中存在独立公式或行内公式，必须使用大模型本身的原生视觉能力，直接读成标准的 LaTeX 语法（`$ $` 或 `$$ $$`）。
3. **插图识别**：如果页面中存在论文原图或表格图，不要丢失！请使用标准的 Markdown 图片语法把它插进去（例如：`![Figure 1: 图表说明](图片占位或者截图路径)`）。
4. **统一汇总**：将这“一页”所得到的所有文本、公式和图片引用，追加写入到**同一个汇总的 `.md` 知识库文件**中。

### 第二步：上下文极限保护与刷新 (Context Degradation Protection)
长篇论文会严重消耗大模型的输出 Token 和记忆窗口，为了确保不丢失排版质量：
1. **进度记忆**：必须在 `task.md` 文件中记录当前读取到了第几页。
2. **主动悬挂 (Suspend)**：评估当前的上下文长度，如果中途发生即将超出文本生成范围或模型记忆极限的情况（例如连续处理了 4-5 页之后），你必须**暂时终止运行**。
3. **刷新衔接**：使用 `notify_user` 通知用户：“进度已保存到 task.md（如第5页），为避免大模型上下文遗忘，请发送‘继续’完成窗口刷新。”。当用户发送第二次请求时，你只需要看一眼 `task.md`，立刻从第 6 页开始接着往下读并追加 md 即可。

### 第三步：格式化编排为目标 Word (Final Typesetting)
当所有 N 张图片都被读取并完全写入汇总的 `.md` 文件后：
1. **加载全文**：大模型去读取这个汇总完成且包含所有正确公式和图片的 `.md` 文件。
2. **排版生成**：按照用户最初指定的格式要求（例如：标题应该有多大、正文什么字体、是否段落缩进等 IEEE 规范），结合大模型对格式的理解，编写并执行生成能够体现此排版的 `.docx` 文件。

## 执行口令指引 (Execution Guide)
用户发送：“按照某某格式排版这个 PDF”，你需要：
1. 切图（放入 `resources/pages/`）。
2. 在 `task.md` 中按页写好 `[ ] Page 1` ..., `[ ] Page N`。
3. 循环调用 `view_file` 看图并输出 markdown 一页一页追加。
4. 觉得上下文不堪重负了，就打断任务并告诉用户让他回复“继续”。
5. 最终调用 `docx` 库或类似工具，将写好的总汇 `.md` 吐出为完美的带格式 `.docx`。
